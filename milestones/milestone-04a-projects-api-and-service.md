# Milestone 04a — Projects API and Service

## Milestone: Projects Cosmos DB service and API routes

> **Validates:** `npm run build` succeeds. `npm start` starts the server. `GET /api/projects` returns HTTP 200 with a JSON array (empty array `[]` when no projects exist). `GET /api/projects/nonexistent-id` returns HTTP 404 with JSON `{ "message": ... }`. `npm test` passes all existing tests.

> **Reference files:** `src/server/services/sampleSpecs.ts` (existing service pattern — async functions, try/catch, descriptive error messages), `src/server/routes/sampleSpecs.ts` (existing route pattern — Router export, thin handlers calling service, JSON error envelope), `src/server/azure/cosmosClient.ts` (shared CosmosClient instance), `src/server/config.ts` (exports `cosmosDatabaseName` and `cosmosContainerName` — use these instead of hardcoded strings), `src/server/app.ts` (where to register new router with `app.use("/api", ...)` before the API 404 handler).
