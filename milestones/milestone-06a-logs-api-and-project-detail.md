# Milestone 06a — Logs API, project detail page, and log viewer

## Milestone: Logs API backend, project detail page, and log viewer component

> **Validates:** GET /api/projects/:id returns 200 with project JSON for an existing project. GET /api/projects/:id/logs returns 200 with a JSON object containing a `lines` array (may be empty). The page at /projects/:id renders with the project name, a "Back to Dashboard" link, and a terminal-style log viewer area. The page at /projects/:id for a nonexistent project shows an error state (not a crash). The app builds with `npm run build` and starts with `npm start`.

> **Reference files:** `src/server/services/sampleSpecs.ts` (service pattern for blob storage reads), `src/server/routes/sampleSpecs.ts` (route pattern with error handling), `src/server/routes/projects.ts` (existing projects router to extend), `src/client/pages/SampleSpecsPage.tsx` (page pattern with fetch, loading, error states), `src/client/pages/ProjectDetailPage.tsx` (placeholder to replace), `src/server/app.ts` (router registration), `src/server/azure/blobClient.ts` (shared blob client)

## Tasks

- [x] Create `src/server/services/logsService.ts` with a `getProjectLogs(projectId: string)` function that uses `blobServiceClient.getContainerClient(projectId)` to list blobs matching `*.log` and `events.jsonl`, downloads each blob's content, splits into lines, and returns a flat `string[]` of all log lines sorted chronologically — follow the same blob download pattern as `getSpecContent` in `sampleSpecs.ts` (stream chunks into Buffer, convert to UTF-8), cap at 5000 lines to avoid unbounded memory
- [x] Add `GET /api/projects/:id/logs` route in `src/server/routes/projects.ts` that calls `getProjectById(id)` first (return 404 if not found), then calls `getProjectLogs(id)` and returns `{ lines: string[] }` — return `{ lines: [] }` if the blob container does not exist (catch `ContainerNotFound` by checking Azure SDK RestError statusCode, not string matching), return 500 with `{ message }` for other errors
- [x] Build the project detail page header in `src/client/pages/ProjectDetailPage.tsx`: fetch project from `GET /api/projects/:id` on mount, display project name as `<h1>`, `createdAt` formatted as a readable date in muted text below the heading, and a "Back to Dashboard" `<Link to="/">` above the heading — show loading skeleton while fetching, show error message with retry button if fetch fails, show "Project not found" for 404 responses
- [ ] Create `src/client/components/LogViewer.tsx` — a terminal-style log display component that accepts `lines: string[]`, `loading: boolean`, and `error: string | null` props, renders lines in a scrollable `<div>` with `bg-zinc-900 text-zinc-100 font-mono text-sm rounded-lg border border-zinc-700 p-4` styling, shows "Loading logs..." with a spinner when loading is true and lines are empty, shows "No logs yet — logs will appear here when a build runs." when not loading and lines are empty, shows error message with retry button (via an `onRetry` prop) when error is set
- [ ] Add auto-scroll behavior to `LogViewer.tsx`: use a `useRef` on the scroll container and a `useEffect` that scrolls to bottom when `lines` changes — detect manual scroll-up by comparing `scrollTop + clientHeight` to `scrollHeight` (with a small threshold like 50px), set an `isAtBottom` state flag, only auto-scroll when `isAtBottom` is true, resume auto-scroll when user scrolls back to the bottom
- [ ] Add polling with pause/resume to `ProjectDetailPage.tsx`: use `setInterval` to fetch `GET /api/projects/:id/logs` every 5 seconds, store lines in state, pass to `LogViewer` — add a `paused` state boolean toggled by a button, skip fetch when paused, clear interval on unmount, do not show loading indicator on poll refreshes (only on initial load)
- [ ] Create pause/resume toggle button inside `LogViewer.tsx` (top-right corner, positioned with `absolute` inside a `relative` container): show "Pause" with a `Pause` Lucide icon and a green pulsing dot (`animate-pulse bg-green-500 rounded-full w-2 h-2`) when polling is active, show "Resume" with a `Play` Lucide icon and a grey dot (`bg-zinc-500`) when paused — accept `paused: boolean` and `onTogglePause: () => void` props
