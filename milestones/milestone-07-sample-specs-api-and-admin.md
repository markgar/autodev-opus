# Milestone 07 — Sample Specs API and Admin Page

## Milestone: Sample specs blob storage service, CRUD API routes, and admin page UI

> **Validates:** `npm run build` succeeds. `npm start` starts the server. `GET /api/sample-specs` returns HTTP 200 with a JSON array (empty if no specs exist). `POST /api/sample-specs` with JSON body `{ "name": "test.md", "content": "# Hello" }` returns HTTP 201. `GET /api/sample-specs/test.md` returns HTTP 200 with text content `# Hello`. `DELETE /api/sample-specs/test.md` returns HTTP 204. After delete, `GET /api/sample-specs` returns an empty array. Navigating to `/admin/sample-specs` renders a page with heading "Sample Specs" and an "Upload" button. The page shows "No sample specs uploaded yet" empty state when no specs exist. `npm test` passes all existing tests.

> **Reference files:** `src/server/routes/health.ts` (existing route pattern — Router export, mounted under /api), `src/server/azure/blobClient.ts` (shared BlobServiceClient instance to use for blob operations), `src/server/app.ts` (where to register new router with app.use), `src/client/pages/SampleSpecsPage.tsx` (placeholder page to replace with full implementation), `src/client/components/AppSidebar.tsx` (existing component pattern — functional component with Tailwind classes), `src/client/index.css` (theme CSS variables to extend).
